# 0.1.1 — 2026-05-25

## Fixed

- **`epa_get_tri_releases` / `epa_search_tri_releases`** — corrected field name mappings from `tri_reporting_form`; chemical names and release quantities (air, water, land, underground injection) were always empty due to wrong column names ([#3](https://github.com/cyanheads/epa-mcp-server/issues/3))
- **`epa_search_superfund`** — `site_name` was always empty; fixed wrong field name in `normalizeSemsSite()` ([#4](https://github.com/cyanheads/epa-mcp-server/issues/4))
- **`epa://superfund/{site_id}` resource** — site lookup heuristic was wrong; resource never found sites by ID; fixed ID matching logic ([#6](https://github.com/cyanheads/epa-mcp-server/issues/6))
- **`epa_search_water_systems`** — `has_violation` filter crashed because `violation_flag` column does not exist in SDWIS; switched to correct column; city field was always missing and is now populated ([#5](https://github.com/cyanheads/epa-mcp-server/issues/5))
- **`epa_get_air_quality`** — cache key generation produced invalid characters (slashes from lat/lng), crashing every call; replaced with a safe `key-value` join ([#1](https://github.com/cyanheads/epa-mcp-server/issues/1))
- **`epa_search_facilities` / `epa_search_violations`** — ECHO API bot-blocking returned empty results silently; added `User-Agent` header to all ECHO requests ([#2](https://github.com/cyanheads/epa-mcp-server/issues/2))
- **`AIRNOW_API_KEY`** — env var was missing from `server.json` `environmentVariables` and `manifest.json` `user_config`; added to both ([#7](https://github.com/cyanheads/epa-mcp-server/issues/7))
- **`epa_get_facility`** — tool description no longer leaks internal "ECHO DFR endpoints in parallel" implementation detail ([#8](https://github.com/cyanheads/epa-mcp-server/issues/8))
