# 0.1.5 — 2026-06-12

## Added

- **TRI truncation signal** — `epa_search_tri_releases` emits an `enrichment` block (`truncated`, `shown`, `cap`) via `ctx.enrich` when results hit the requested `limit`, flagging that more records may exist.
- **Plugin manifests** — `.claude-plugin/plugin.json` and `.codex-plugin/` manifests for one-step install in Claude Code and Codex.
- **Docker `HEALTHCHECK`** — bun-native `fetch` against `/healthz` (slim image ships no curl/wget), plus OCI `version` and `source` labels driven by an `APP_VERSION` build arg.
- **`format:unsafe`** script (`biome check --write --unsafe`); `format` no longer applies unsafe fixes by default. `stdio` and `streamable-http` added to package keywords.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9` → `^0.10.6` — framework adoption.
- **Validation error codes** — the geographic/location precondition errors across `epa_search_facilities`, `epa_search_violations`, `epa_search_water_systems`, `epa_search_superfund`, `epa_search_tri_releases`, and `epa_get_air_quality` now use `JsonRpcErrorCode.ValidationError` (renamed from `InvalidParams` in 0.10.x).
- **Server identity** — `createApp()` sets `name` and `title` explicitly to `epa-mcp-server`.

## Dependency bumps (dev)

- `@types/node` ^25.9.2 → ^25.9.3
